# CITYARRAY Sovereign Architecture

## City-Controlled Edge-to-Cloud Infrastructure

---

## Executive Summary

CITYARRAY's Sovereign Architecture ensures that cities maintain complete control over their data while benefiting from AI-powered emergency communication. All sensitive data remains within the city's infrastructure, with optional cloud connectivity for updates and anonymized analytics only.

**Key Principles:**
- **Data sovereignty** â€” All PII and sensitive data stays on-premise
- **Edge-first** â€” AI processing happens on device, not in cloud
- **Zero-trust** â€” Every communication is authenticated and encrypted
- **Resilient** â€” Works offline, on mesh, or with full connectivity

**Purpose:** Provide updates, threat intelligence, and support without accessing city data.

| Service | Data Flow | City Data Exposed |
|---------|-----------|-------------------|
| Software updates | Cloud â†’ City | None |
| AI model updates | Cloud â†’ City | None |
| Threat intelligence | Cloud â†’ City | None |
| Anonymized analytics | City â†’ Cloud (opt-in) | Aggregate counts only |
| Support diagnostics | City â†’ Cloud (on request) | Sanitized logs only |

- All communications TLS 1.3 encrypted
- Updates cryptographically signed
- City can operate 100% offline if desired
- No inbound connections from cloud to city

#### Database Schema (Core Tables)

```sql
-- Devices
CREATE TABLE devices (
    id UUID PRIMARY KEY,
    name VARCHAR(100),
    zone_id UUID REFERENCES zones(id),
    status VARCHAR(20),
    last_heartbeat TIMESTAMP,
    firmware_version VARCHAR(20),
    ip_address INET,
    config JSONB,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- Alerts
CREATE TABLE alerts (
    id UUID PRIMARY KEY,
    tier VARCHAR(20), -- informational, advisory, warning, emergency
    message JSONB, -- {en: "...", es: "...", ...}
    zones UUID[], -- target zones
    created_by UUID REFERENCES users(id),
    authorized_by UUID REFERENCES users(id),
    authorization_method VARCHAR(50),
    started_at TIMESTAMP,
    ended_at TIMESTAMP,
    created_at TIMESTAMP
);

-- Audit Log
CREATE TABLE audit_log (
    id BIGSERIAL PRIMARY KEY,
    timestamp TIMESTAMP DEFAULT NOW(),
    user_id UUID,
    action VARCHAR(100),
    resource_type VARCHAR(50),
    resource_id UUID,
    details JSONB,
    ip_address INET,
    signature VARCHAR(256) -- cryptographic proof
);

-- Emergency Protocols
CREATE TABLE emergency_protocols (
    id UUID PRIMARY KEY,
    name VARCHAR(100),
    tier VARCHAR(20),
    trigger_conditions JSONB,
    actions JSONB, -- ordered list of actions
    authorization_required VARCHAR(50),
    timeout_seconds INT,
    active BOOLEAN,
    created_at TIMESTAMP
);
```

### Audit Logging

All actions are logged with:

```json
{
  "id": "log_123456",
  "timestamp": "2024-12-30T15:30:00Z",
  "user_id": "user_789",
  "user_email": "operator@city.gov",
  "action": "EMERGENCY_ALERT_CREATED",
  "resource": "alert_456",
  "details": {
    "tier": "emergency",
    "zones": ["zone_a", "zone_b"],
    "message_preview": "EVACUATE NOW..."
  },
  "ip_address": "10.0.0.50",
  "user_agent": "Mozilla/5.0...",
  "authorization_method": "mfa_totp",
  "signature": "ed25519_signature_of_log_entry"
}
```

Logs are:
- Cryptographically signed (tamper-evident)
- Retained per city policy (default 7 years)
- Exportable for legal/audit purposes
- Never transmitted to CITYARRAY cloud

- [ ] Firewall rules defined
- [ ] VPN/private APN configured
- [ ] Server hardware provisioned
- [ ] SSL certificates generated
- [ ] User accounts created
- [ ] Emergency protocols defined
- [ ] Integration points identified

### Server Deployment

- [ ] Control server software installed
- [ ] Database initialized
- [ ] MQTT broker configured
- [ ] API gateway configured
- [ ] SSL certificates installed
- [ ] Backup system configured
- [ ] Monitoring configured

### Zone Gateway Deployment

- [ ] Hardware installed
- [ ] Network connectivity verified
- [ ] Certificates provisioned
- [ ] LoRaWAN configured (if applicable)
- [ ] Failover tested

### Edge Device Deployment

- [ ] Physical installation complete
- [ ] Network connectivity verified
- [ ] Device registered with server
- [ ] Display tested
- [ ] Audio tested
- [ ] Camera tested
- [ ] AI detection verified
- [ ] Mesh connectivity verified

- [ ] Failover test (gateway offline)

- [ ] Performance baseline established
- [ ] Staff training completed
- [ ] Documentation delivered
- [ ] Support handoff complete
